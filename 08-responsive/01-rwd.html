C'est quoi le Responsive ?

Le responsive web design est une technique, permettant de créer un design qui s'adapte au device utilisé (téléphone, tablette, ordi...).
Il existe plusieurs manières de faire :

  • Faire une version pour mobile, une autre version pour tablette... mais lourd en cas de modification
  • Utiliser les Media queries : permet de faire une condition en fonction de la taille du device et d'adapter le CSS en fonction (Exemple)

Dimensions minimales et maximales

Il est possible de définir des largeurs et hauteurs minimales et maximales 

.container{
	max-width: 1200px; /* Si la résolution est <1200px, l'élément va s'adapter, sinon il sera limité à 1200px */
}

.post-resume{
	min-height: 300px; /* L'élément aura une hauteur minimale de 300px, même si la hauteur de son contenu est < */
}

Les media queries

Les media queries permettent de spécifier un média pour une (ou plusieurs) feuille de style. Ex : imprimante (print), screen (écran)...
Pour spécifier ces informations, on ajoute l'attribut media à l'appel du CSS (link), ou directement dans le fichier CSS.

Un fichier CSS par condition

Assigner un fichier CSS selon les conditions (largeur de l'écran, hauteur du navigateur, oriention du téléphone...).
Exemple : le fichier mobile.css ne sera appelé que si la résolution du navigateur est <= 500px 

<link rel="stylesheet" type="text/css" media="screen and (max-width:500px)" href="mobile.css" />

Ajouter les conditions directement dans le CSS

/* CSS ne s'appliquent que si on est sur une résolution <= 500px */
@media screen and (max-width:500px){
	body{
		background: blue;
	}
}

Principe du mobile first

Le principe du mobile first consiste à commencer par le style en format mobile, puis d'augmenter la résolution. 

body{
	background: blue;
}

/* CSS ne s'appliquent que si on est sur une résolution > 800px */
@media screen and (min-width:800px){
	body{
		background: red;
	}
}

Viewport - Documentation

La balise meta viewport (à positionner dans le <head>) permet de contrôler l'apparence de l'affichage de la page en fonction des devices.
Par défaut, le navigateur mobile va afficher le contenu comme si on était sur Desktop.
Il va donc falloir préciser comment le navigateur doit afficher la page web.

De nombreuses propriétés peuvent être définies, séparées par des virgules.
Dans ce cas, on définit la largeur de la fenêtre (width) comme étant celle du device (tablette, smartphone...), et on définit l'échelle de la fenêtre à 1. 

<meta name="viewport" content="width=device-width, initial-scale=1.0" />
