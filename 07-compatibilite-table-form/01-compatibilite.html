Préfixes CSS - Générer le CSS avec les préfixes

Les nouvelles propriétés CSS3 (border-radius, box-shadow ...) mettent du temps à être normalisées. Du coup, sous autorisation du W3C, les éditeurs de navigateurs (vendors) ont la possibilité d'introduire le support de propriétés spécifiques : les préfixes vendor, en attendant la sortie d'une nouvelle version du standard (W3C).
Aujourd'hui, la majorité des navigateurs comprenent les propriétés sans les préfixes, mais il faut rester prudent...

Les principaux préfixes :

  • -webkit- (pour Chrome, Safari,…)
  • -moz- (pour Firefox, Flock,…)
  • -o- (pour Opera)
  • -ms- (pour Internet Explorer)
.elem{
	box-sizing: border-box;
}
Devient 
.elem{
	-webkit-box-sizing: border-box;
	   -moz-box-sizing: border-box;
	        box-sizing: border-box;
}

Utilisation d'un plugin javascript - Prefixfree : prefixfree - Avantages & inconvénients

Prefixfree est un plugin JS qui détecte le navigateur utilisé et ajoute automatiquement le préfix correspondant à votre CSS.
Pour l'utiliser, il suffit de le télécharger et de l'importer dans votre fichier HTML : 

<script type="text/javascript" src="scripts/prefixfree.min.js"></script>
NB : Ne vous embêtez pas à les ajouter durant la formation, mais il faut y penser quand un site est en production !

Compatibilité HTML5

La plupart des vieux navigateurs ne reconnaissent pas les éléments HTML5 (il seront identifiés comme étant de type "en-ligne" par défaut, ce que l'on peut comparer à un <span>), mais rien ne nous empêche de les utiliser en connaissance de cause en leur appliquant un display approprié (display:block ou autre).
Attention Le hic : Internet Explorer 8 et versions inférieures nécessitent un code JavaScript afin de créer au préalable ces éléments dans leur DOM sous peine de ne rien afficher du tout.

Utilisation d'un plugin javascript - HTML5shiv : HTML5shiv - Avantages & inconvénients

html5shim (ou html5shiv), offre la possibilité aux anciennes versions d'IE de reconnaître et d'interpréter les éléments HTML5, à l'aide d'un simple code HTML/JS à placer dans le head de votre HTML : 

<!--[if lt IE 9]>
<script type="text/javascript" src="scripts/html5shiv.min.js"></script>
<![endif]-->
NB : Ne vous embêtez pas à les ajouter durant la formation, mais il faut y penser quand un site est en production !
NB2 : Le fichier reset.css redéfinit déjà les balises HTML5 comme des block ;)
