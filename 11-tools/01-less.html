Un pré-processeur ? Késako ?

Depuis quelques années, c'est la mode des pré-processeurs CSS. Ils peuvent être considérés comme des outils permettant de générer dynamiquement du CSS.

Les pré-processeurs interviennent en amont, avant la rédaction de CSS, puisque ce sont eux qui vont générer la feuille de style à partir d'un langage spécifique à chaque pré-processeur.

Les principaux pré-processeurs du marché sont :

  • LESS : que nous allons voir car plus simple d'implémentation que les autres
  • SASS : selon moi le plus abouti et celui que j'utilise
  • Stylus : inconnu au bataillon, mais peut être que vous serez curieux !
NB : peu importe le pré-processeur, la logique restera quasiment la même, seule la syntaxe sera différente.

Pourquoi un pré-processeur ?

On commence doucement à se rendre compte que :

  • L'indentation est importante pour grouper visuellement le CSS
  • Les propriétés / valeurs sont souvent répétées, ou sont regroupées pour éviter les redondances
  • Les éléments récurents d'un design (font-family, color, background, tailles) vont être utilisés à plusieurs reprises, et ne seront pas pratique à maintenir
  • Les prefixes pour les propriétés CSS3 alourdissent le code
  • ...

Les pré-processeurs, vont par exemple permettre de :

  • Définir des variables
  • Imbriquer des propriétés
  • Préfixer automatiquement les propriétés CSS3

Less - comment ça marche ?

Désormais, vous n’allez plus écrire des feuilles CSS mais des feuilles LESS. Malheureusement, le navigateur ne comprend pas le LESS. Il va donc falloir convertir le LESS en CSS.

Déclarer un fichier LESS :

<link rel="stylesheet/less" type="text/css" href="styles/style.less" />

Compiler le LESS en CSS (client side) - Télécharger less.min.js :

less.js est un fichier javascript compilant le LESS en CSS au chargement de la page. Il faut l'indiquer dans le head, après votre link less

<link rel="stylesheet/less" type="text/css" href="styles/style.less" />
<script type="text/javascript" src="scripts/less.js"></script>

Les variables - Documentation

CSS classique :

a{
	color: #333;
}

.intro{
	background: #333;
	color: #fff;
}

LESS :

@gray: #333; /* on définit une variable gray qui a comme valeur #333 */

a{
	color: @gray;
}

.intro{
	background: @gray;
	color: #fff;
}

Imbrication / hiérarchisation de propriétés

CSS classique :

.nav{
	background: #333;
	padding: 10px;
}

.nav ul{
	margin: 0;
	padding: 0;
	font-size: 0;
}

.nav ul li{
	display: inline-block;
	vertical-align: middle;
	font-size: 1rem;
}

.nav ul li a{
	text-decoration: none;
	text-transform: uppercase;
}

.nav ul li a:hover{
	color: red;
}

LESS :

.nav{
	background: #333;
	padding: 10px;
	
	ul{
		margin: 0;
		padding: 0;
		font-size: 0;
	
		li{
			display: inline-block;
			vertical-align: middle;
			font-size: 1rem;
	
			a{
				text-decoration: none;
				text-transform: uppercase;

				&:hover{
					color: red;
				}
			}
		}
	}
}
& correspond à l'élément courant

Les mixins

Les mixins sont comme des variables, mais peuvent contenir plusieurs propriétés/valeurs.

CSS classique :

.intro{
	-webkit-border-radius: 12px;
	-moz-border-radius: 12px;
	border-radius: 12px;
}

.sidebar{
	-webkit-border-radius: 12px;
	-moz-border-radius: 12px;
	border-radius: 12px;
}

LESS :

.radius{
     -webkit-border-radius: 12px;
     -moz-border-radius: 12px;
     border-radius: 12px;
}

.intro{
	.radius;
}

.sidebar{
	.radius;
}

Les mixins - avec paramètres

CSS classique :

.intro{
	-webkit-border-radius: 15px;
	-moz-border-radius: 15px;
	border-radius: 15px;
}

.sidebar{
	-webkit-border-radius: 50px;
	-moz-border-radius: 50px;
	border-radius: 50px;
}

LESS :

.radius(@radius){
     -webkit-border-radius: @radius;
     -moz-border-radius: @radius;
     border-radius: @radius;
}

.intro{
	.radius(15px);
}

.sidebar{
	.radius(50px);
}

Les opérations

CSS classique :

"Pas possible..." (en fait si, il existe une fonction calc() depuis CSS3 - Informations)

LESS :

@sidebarWidth: 300px;

.sidebar{
	width: @sidebarWidth;
	float: left;
}
.content{
	width: 100% - @sidebarWidth;
	float: left;
}
